## 1. 背景与目标

为 Commute 项目设计 PostgreSQL 数据库 Schema，支持：
- 单/多家庭住址管理
- AI 公司推荐缓存
- 公司管理与状态追踪
- 通勤计算结果缓存
- 历史查询快照
- **预留多用户扩展能力**（当前为单虚拟用户）

## 2. 设计原则

### 2.1 命名规范
- 表名：`snake_case`，**单数形式**（`user` 而非 `users`，但 `user` 是保留字 → 用 `app_user`）
- 字段名：`snake_case`
- 主键：统一 `id`，类型 `BIGSERIAL`
- 外键：`{表名}_id`（如 `user_id`、`company_id`）
- 时间戳：`created_at`、`updated_at`，类型 `TIMESTAMPTZ`，默认 `NOW()`
- 软删除：业务表统一加 `deleted_at TIMESTAMPTZ NULL`

### 2.2 通用字段
所有业务表必须包含：
```sql
id          BIGSERIAL PRIMARY KEY,
user_id     BIGINT NOT NULL DEFAULT 1 REFERENCES app_user(id),
created_at  TIMESTAMPTZ NOT NULL DEFAULT NOW(),
updated_at  TIMESTAMPTZ NOT NULL DEFAULT NOW(),
deleted_at  TIMESTAMPTZ
```

### 2.3 坐标存储
- 经纬度统一使用 **`NUMERIC(10, 7)`**
  - 7 位小数 ≈ 1 cm 精度，足够通勤场景
  - 不用 `FLOAT/DOUBLE`：避免精度漂移
- 坐标系：**GCJ-02（高德坐标系）**，所有表统一
- 不引入 PostGIS（MVP 不需要空间查询）

### 2.4 枚举处理
- 使用 PostgreSQL 原生 `ENUM` 类型（值固定且数量少时）
- 未来可能扩展的字段用 `VARCHAR + CHECK` 约束

### 2.5 JSON 字段
- 高德/豆包返回的复杂结构用 `JSONB`
- 不做精细化 schema 拆分，保持灵活性
- 关键字段冗余成列（便于查询和索引）

### 2.6 索引策略
- 所有外键自动加索引
- 高频查询字段加索引（如 `user_id + deleted_at`）
- 缓存表加联合索引覆盖命中条件

### 2.7 时区
- 统一 `TIMESTAMPTZ`（UTC 存储 + 时区感知）
- 应用层默认时区：`Asia/Shanghai`

## 4. 详细 Schema

### 4.1 `app_user` —— 用户表（预留）

```sql
CREATE TABLE app_user (
    id          BIGSERIAL PRIMARY KEY,
    name        VARCHAR(64),
    phone       VARCHAR(20) UNIQUE,
    email       VARCHAR(128) UNIQUE,
    avatar_url  VARCHAR(512),
    created_at  TIMESTAMPTZ NOT NULL DEFAULT NOW(),
    updated_at  TIMESTAMPTZ NOT NULL DEFAULT NOW(),
    deleted_at  TIMESTAMPTZ
);

-- 初始虚拟用户
INSERT INTO app_user (id, name) VALUES (1, '默认用户');
SELECT setval('app_user_id_seq', 1, true);
```

> **说明**：MVP 阶段所有数据 `user_id = 1`。未来接入登录后，新建用户记录并迁移数据归属。

### 4.2 `user_profile` —— 用户画像

```sql
CREATE TYPE company_type_enum AS ENUM ('big_tech', 'mid_tech', 'startup', 'foreign', 'other');

CREATE TABLE user_profile (
    id                  BIGSERIAL PRIMARY KEY,
    user_id             BIGINT NOT NULL DEFAULT 1 REFERENCES app_user(id),
    current_city        VARCHAR(64) NOT NULL,        -- 如 "北京"
    current_city_code   VARCHAR(16),                 -- 高德 adcode，如 "110000"
    target_position     VARCHAR(128) NOT NULL,       -- 如 "后台开发"
    experience_years    SMALLINT,                    -- 0-30
    preferred_company_types company_type_enum[],     -- 多选
    created_at          TIMESTAMPTZ NOT NULL DEFAULT NOW(),
    updated_at          TIMESTAMPTZ NOT NULL DEFAULT NOW(),
    UNIQUE (user_id)
);
```

> **决策**：单用户单 profile，用 UNIQUE 约束。修改即更新。

### 4.3 `home_address` —— 家庭住址

```sql
CREATE TABLE home_address (
    id            BIGSERIAL PRIMARY KEY,
    user_id       BIGINT NOT NULL DEFAULT 1 REFERENCES app_user(id),
    alias         VARCHAR(64) NOT NULL,              -- 用户自定义，如 "我家"
    address       VARCHAR(512) NOT NULL,             -- 高德标准化地址
    province      VARCHAR(32),
    city          VARCHAR(32),
    district      VARCHAR(32),
    longitude     NUMERIC(10, 7) NOT NULL,
    latitude      NUMERIC(10, 7) NOT NULL,
    is_default    BOOLEAN NOT NULL DEFAULT FALSE,
    note          TEXT,                              -- 用户备注
    created_at    TIMESTAMPTZ NOT NULL DEFAULT NOW(),
    updated_at    TIMESTAMPTZ NOT NULL DEFAULT NOW(),
    deleted_at    TIMESTAMPTZ
);

CREATE INDEX idx_home_address_user ON home_address (user_id) WHERE deleted_at IS NULL;

-- 同一用户只能有一个默认地址（部分唯一索引）
CREATE UNIQUE INDEX uniq_home_address_default
    ON home_address (user_id)
    WHERE is_default = TRUE AND deleted_at IS NULL;
```

> **决策**：用部分唯一索引保证"同时只有一个默认"，比触发器更简洁可靠。

### 4.4 `company` —— 公司

```sql
CREATE TYPE company_status_enum AS ENUM (
    'watching',     -- 关注
    'applied',      -- 已投递
    'interviewing', -- 面试中
    'offered',      -- 已 offer
    'rejected',     -- 已拒/被拒
    'archived'      -- 归档
);

CREATE TYPE company_source_enum AS ENUM (
    'ai_recommend', -- AI 推荐
    'manual'        -- 手动添加
);

CREATE TABLE company (
    id            BIGSERIAL PRIMARY KEY,
    user_id       BIGINT NOT NULL DEFAULT 1 REFERENCES app_user(id),
    name          VARCHAR(128) NOT NULL,
    address       VARCHAR(512) NOT NULL,
    province      VARCHAR(32),
    city          VARCHAR(32),
    district      VARCHAR(32),
    longitude     NUMERIC(10, 7) NOT NULL,
    latitude      NUMERIC(10, 7) NOT NULL,
    category      company_type_enum,                 -- 复用 user_profile 的枚举
    industry      VARCHAR(64),                       -- 如 "互联网"
    status        company_status_enum NOT NULL DEFAULT 'watching',
    source        company_source_enum NOT NULL DEFAULT 'manual',
    ai_reason     TEXT,                              -- AI 推荐时保存的理由
    note          TEXT,                              -- 用户备注（薪资、HR 等）
    created_at    TIMESTAMPTZ NOT NULL DEFAULT NOW(),
    updated_at    TIMESTAMPTZ NOT NULL DEFAULT NOW(),
    deleted_at    TIMESTAMPTZ
);

CREATE INDEX idx_company_user ON company (user_id) WHERE deleted_at IS NULL;
CREATE INDEX idx_company_user_status ON company (user_id, status) WHERE deleted_at IS NULL;

-- 同用户下公司名 + 地址 不重复（防止 AI 重复推荐）
CREATE UNIQUE INDEX uniq_company_user_name_addr
    ON company (user_id, name, address)
    WHERE deleted_at IS NULL;
```

### 4.5 `commute_query` —— 通勤查询会话

> **作用**：每次用户发起的"批量通勤计算"是一次查询，对应 1 个 home + N 个 company + M 种出行方式。

```sql
CREATE TYPE transport_mode_enum AS ENUM (
    'transit',  -- 公交/地铁
    'driving',  -- 驾车
    'cycling',  -- 骑行
    'walking'   -- 步行
);

CREATE TYPE time_strategy_enum AS ENUM (
    'depart_at',  -- 指定出发时间，推算到达
    'arrive_by'   -- 指定到达时间，推算出发
);

CREATE TABLE commute_query (
    id              BIGSERIAL PRIMARY KEY,
    user_id         BIGINT NOT NULL DEFAULT 1 REFERENCES app_user(id),
    home_id         BIGINT NOT NULL REFERENCES home_address(id),
    transport_modes transport_mode_enum[] NOT NULL,        -- 多选
    -- 早通勤设置
    morning_strategy time_strategy_enum NOT NULL DEFAULT 'depart_at',
    morning_time     TIME NOT NULL DEFAULT '08:00',        -- 出发或到达时间（取决于 strategy）
    -- 晚通勤设置
    evening_strategy time_strategy_enum NOT NULL DEFAULT 'depart_at',
    evening_time     TIME NOT NULL DEFAULT '17:30',
    -- 通用
    weekday          SMALLINT NOT NULL DEFAULT 1,          -- ISO weekday，1=Mon
    buffer_minutes   SMALLINT NOT NULL DEFAULT 5,
    created_at       TIMESTAMPTZ NOT NULL DEFAULT NOW(),
    updated_at       TIMESTAMPTZ NOT NULL DEFAULT NOW()
);

CREATE INDEX idx_commute_query_user ON commute_query (user_id, created_at DESC);
```

### 4.6 `commute_result` —— 通勤计算结果（含缓存）

> **作用**：单条 [home × company × mode × direction] 的计算结果。既是查询关联表，也是 7 天缓存。

```sql
CREATE TYPE commute_direction_enum AS ENUM ('to_work', 'to_home');

CREATE TABLE commute_result (
    id              BIGSERIAL PRIMARY KEY,
    user_id         BIGINT NOT NULL DEFAULT 1 REFERENCES app_user(id),
    query_id        BIGINT REFERENCES commute_query(id),    -- 可空（后台缓存命中也写入）
    home_id         BIGINT NOT NULL REFERENCES home_address(id),
    company_id      BIGINT NOT NULL REFERENCES company(id),
    direction       commute_direction_enum NOT NULL,
    transport_mode  transport_mode_enum NOT NULL,
    -- 时间
    depart_time     TIME NOT NULL,                          -- 实际出发时间
    arrive_time     TIME NOT NULL,                          -- 实际到达时间
    weekday         SMALLINT NOT NULL DEFAULT 1,
    -- 计算结果
    duration_min    INT NOT NULL,                           -- 含 buffer
    duration_min_raw INT NOT NULL,                          -- 不含 buffer（高德原值）
    distance_km     NUMERIC(8, 2) NOT NULL,
    cost_yuan       NUMERIC(8, 2),                          -- 票价/油费估算
    transfer_count  SMALLINT,                               -- 换乘次数（仅 transit 有意义）
    route_detail    JSONB NOT NULL,                         -- 高德返回完整路线
    -- 缓存控制
    calculated_at   TIMESTAMPTZ NOT NULL DEFAULT NOW(),
    expires_at      TIMESTAMPTZ NOT NULL,                   -- 默认 calculated_at + 7 days
    -- 错误处理
    is_failed       BOOLEAN NOT NULL DEFAULT FALSE,
    error_message   TEXT,
    created_at      TIMESTAMPTZ NOT NULL DEFAULT NOW(),
    updated_at      TIMESTAMPTZ NOT NULL DEFAULT NOW()
);

-- 缓存命中索引（覆盖关键字段）
CREATE INDEX idx_commute_result_cache
    ON commute_result (home_id, company_id, transport_mode, direction, depart_time, expires_at);

CREATE INDEX idx_commute_result_query ON commute_result (query_id);
CREATE INDEX idx_commute_result_user ON commute_result (user_id, calculated_at DESC);
```

> **缓存命中逻辑**：`SELECT ... WHERE home_id=? AND company_id=? AND transport_mode=? AND direction=? AND depart_time=? AND expires_at > NOW() AND is_failed=false ORDER BY calculated_at DESC LIMIT 1`

### 4.7 `ai_recommendation_cache` —— AI 推荐结果缓存

```sql
CREATE TABLE ai_recommendation_cache (
    id                BIGSERIAL PRIMARY KEY,
    user_id           BIGINT NOT NULL DEFAULT 1 REFERENCES app_user(id),
    -- 命中条件（构成缓存键）
    city              VARCHAR(64) NOT NULL,
    position          VARCHAR(128) NOT NULL,
    experience_years  SMALLINT,
    company_types     company_type_enum[],
    cache_key         VARCHAR(256) NOT NULL,                 -- 上述字段拼接的哈希/字符串
    -- 数据
    raw_response      JSONB NOT NULL,                        -- 豆包原始返回
    company_count     SMALLINT NOT NULL,
    -- 缓存控制
    requested_at      TIMESTAMPTZ NOT NULL DEFAULT NOW(),
    expires_at        TIMESTAMPTZ NOT NULL,                  -- requested_at + 24h
    -- 计费追踪
    token_input       INT,
    token_output      INT,
    created_at        TIMESTAMPTZ NOT NULL DEFAULT NOW()
);

CREATE INDEX idx_ai_cache_key ON ai_recommendation_cache (cache_key, expires_at);
CREATE INDEX idx_ai_cache_user ON ai_recommendation_cache (user_id, created_at DESC);
```

> **cache_key 生成**：`md5(user_id || ':' || city || ':' || position || ':' || COALESCE(experience_years, 0) || ':' || array_to_string(company_types, ','))`

### 4.8 `query_history` —— 查询历史快照

> **作用**：保留某次完整查询的"现场"，便于一键恢复。

```sql
CREATE TABLE query_history (
    id           BIGSERIAL PRIMARY KEY,
    user_id      BIGINT NOT NULL DEFAULT 1 REFERENCES app_user(id),
    title        VARCHAR(128),                             -- 用户自定义或默认 "2026-04-15 朝阳-后台开发"
    query_id     BIGINT REFERENCES commute_query(id),
    -- 完整快照（JSON 形式，反序列化后可还原 UI 状态）
    snapshot     JSONB NOT NULL,
    /* snapshot 结构示例：
    {
      "home": { "id": 1, "alias": "我家", "lng": 116.40, "lat": 39.90 },
      "companies": [
        { "id": 1, "name": "字节跳动", "lng": 116.30, "lat": 39.97, "status": "watching" }
      ],
      "transport_modes": ["transit", "driving"],
      "morning": { "strategy": "depart_at", "time": "08:00" },
      "evening": { "strategy": "depart_at", "time": "17:30" },
      "results_summary": [
        { "company_id": 1, "mode": "transit", "morning_min": 55, "evening_min": 62 }
      ]
    }
    */
    created_at   TIMESTAMPTZ NOT NULL DEFAULT NOW(),
    deleted_at   TIMESTAMPTZ
);

CREATE INDEX idx_query_history_user ON query_history (user_id, created_at DESC) WHERE deleted_at IS NULL;
```

## 6. 数据生命周期

| 表 | 写入 | 更新 | 清理策略 |
|----|------|------|---------|
| `app_user` | 注册 | 资料编辑 | 软删除 |
| `user_profile` | 首次设置 | 资料编辑 | 跟随用户 |
| `home_address` | 用户新增 | 编辑 | 软删除 |
| `company` | AI 勾选/手动 | 状态变更 | 软删除 |
| `commute_query` | 每次查询 | 不更新 | 物理删除（关联 result 用 ON DELETE SET NULL） |
| `commute_result` | 计算时 | 仅 expires_at | 定时清理过期 7 天以上的失败记录 |
| `ai_recommendation_cache` | AI 调用时 | 不更新 | 定时清理过期 7 天以上的记录 |
| `query_history` | 用户保存或自动 | 标题编辑 | 软删除，可手动恢复 |

### 6.1 定时清理任务
```sql
-- 每天凌晨执行
DELETE FROM commute_result WHERE expires_at < NOW() - INTERVAL '7 days' AND is_failed = TRUE;
DELETE FROM ai_recommendation_cache WHERE expires_at < NOW() - INTERVAL '7 days';
```

## 8. 风险与权衡

| 决策 | 权衡 | 备注 |
|------|------|------|
| 用 `BIGSERIAL` 不用 `UUID` | 性能/可读性 vs 全局唯一 | MVP 单库，无需 UUID |
| 不引入 PostGIS | 简化 vs 空间查询能力 | 当前仅"点对点"路径，无范围查询需求 |
| `ENUM` 类型 vs `VARCHAR + CHECK` | 可读性 vs 扩展性 | 对相对稳定的枚举用 ENUM；扩展时用 `ALTER TYPE ADD VALUE` |
| 软删除 `deleted_at` | 数据可恢复 vs 查询复杂度 | 所有查询统一带 `WHERE deleted_at IS NULL`，用部分索引规避性能损失 |
| 缓存表与业务表合并 (`commute_result`) | 简化模型 vs 表膨胀 | 单表 7 天 TTL，可控；定时清理 |

## 9. 后续可能扩展

- **多租户**：增加 `tenant_id`，所有索引前置
- **路径详情压缩**：`route_detail` 体积大时启用 `JSONB → toast 压缩`
- **PostGIS**：如需做"附近公司"功能，启用并加 `geography(POINT, 4326)` 字段
- **物化视图**：对比视图聚合数据可建 MV 加速
